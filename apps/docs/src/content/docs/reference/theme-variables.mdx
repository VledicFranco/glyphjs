This page documents all CSS custom properties (variables) available in the GlyphJS theming system. Variables are organized by category and include light/dark theme defaults, descriptions, and which components use them.

For an overview of how theming works, see the [Theming guide](/glyphjs/reference/theming/).

---

## Examples

### Customizing Theme Globally

Set CSS variables on a wrapper element or use the `ThemeProvider` with a custom theme object:

```tsx
import { ThemeProvider } from '@glyphjs/runtime';

const myTheme = {
  name: 'corporate',
  variables: {
    '--glyph-bg': '#ffffff',
    '--glyph-text': '#333333',
    '--glyph-accent': '#0066cc',
    '--glyph-accent-hover': '#0052a3',
    '--glyph-border': '#e0e0e0',
    '--glyph-surface': '#f5f5f5',
    // ... other variables
  },
};

function App() {
  return (
    <ThemeProvider theme={myTheme}>
      {/* All GlyphJS components will use these variables */}
    </ThemeProvider>
  );
}
```

### Customizing Individual Components

Override specific variables using CSS selectors:

```css
/* Make all tables use a different header color */
.my-app {
  --glyph-table-header-bg: #1e3a5f;
  --glyph-table-header-color: #ffffff;
}

/* Style callouts differently */
.my-app {
  --glyph-callout-info-bg: #e3f2fd;
  --glyph-callout-info-border: #2196f3;
}
```

Or target specific theme contexts:

```css
[data-glyph-theme="corporate"] {
  --glyph-accent: #0066cc;
  --glyph-kanban-priority-high: #c62828;
}
```

### Dark Mode Setup

Use the built-in dark theme or create a custom dark variant:

```tsx
import { ThemeProvider, darkTheme } from '@glyphjs/runtime';

// Option 1: Use built-in dark theme
<ThemeProvider theme="dark">
  {children}
</ThemeProvider>

// Option 2: Extend dark theme
const customDark = {
  name: 'custom-dark',
  variables: {
    ...darkTheme.variables,
    '--glyph-accent': '#ff6b6b',
    '--glyph-accent-hover': '#ff8787',
  },
};

<ThemeProvider theme={customDark}>
  {children}
</ThemeProvider>
```

### Scoped Component Styling

Apply variables to specific instances using inline styles:

```tsx
<div style={{
  '--glyph-callout-info-bg': '#fff3cd',
  '--glyph-callout-info-border': '#ffc107',
} as React.CSSProperties}>
  <Callout type="info" title="Note">
    This callout has custom colors.
  </Callout>
</div>
```

## Variable Naming Conventions

GlyphJS CSS variables follow these naming patterns:

| Pattern | Example | Usage |
|---|---|---|
| `--glyph-{property}` | `--glyph-bg`, `--glyph-text` | Global design tokens |
| `--glyph-{component}-{property}` | `--glyph-table-border` | Component-specific styling |
| `--glyph-{component}-{variant}-{property}` | `--glyph-callout-info-bg` | Component variant styling |
| `--glyph-{property}-{modifier}` | `--glyph-accent-hover` | State variations |

All variables include sensible fallback values, ensuring components render correctly even without a theme provider.
